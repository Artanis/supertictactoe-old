Rules of the Game

  • The game board is nine Tic-Tac-Toe boards placed in a larger Tic-Tac-Toe board.

  • The first move may be played anywhere.

    Each move after the first is played in the Tic-Tac-Toe board corresponding to the square the previous move played in.

    For example, if Player 1 plays in the top middle board on the bottom left square, then Player 2 must play in the bottom left board.

    If the indicated board is full, then that move may be played anywhere.

  • Getting three in a row in any board claims the square on the large Tic-Tac-Toe board for the winner

    Moves can be made in that board even after it is won, until it is full.

  • Getting three in a row on the large Tic-Tac-Toe board wins the game for that player

    Known Bug: The game does not end when a player wins, and play may continue until the all squares are full, resulting in a reported Cats Game.

PyGTK Application

The PyGTK application is currently a local-only game that grew out of a simple test UI.

Upon opening the application, you can begin play in a new game immediately. If you wish to view a replay or continue an old game, see Saving and loading games.

Starting a new game

To start a new game, select New Game from the Game menu.

Keep in mind that doing this clears the old game. If you wish to keep the old game, save it

top

Saving and loading games

You can save a game (either an in-progress or game or a completed game) for continuation or playback later by choosing Save Game from the Game menu. The save file is simply a JSON object containing the current game state, and all moves made by the players.

You can load a game by choosing Load Game from the Game Menu and choosing a saved game. Depending on the state of the game, it will either be loaded for continued play or as a replay. Currently, An in-progress game will load instantly, but a completed game will enter replay mode.
